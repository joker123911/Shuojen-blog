## 前言

由於太喜歡 shuyu 的作品，加上自己也想深入學習像素畫，因此決定試試看被譽為神級像素軟體的 Aseprite！雖然官方預編譯的版本是收費的，但因為 Aseprite 專案本身開源，官方也提供了自行編譯的指南。

以下是我整理在 Windows 系統下自行編譯 Aseprite 的詳細步驟：

## 編譯前準備

在開始編譯之前，需要安裝與下載以下工具和依賴項目：

1. **Visual Studio 2022 社群版**：
   - 安裝時務必勾選「**使用 C++ 的桌面開發** (Desktop development with C++)」工作負載。
   - 確保包含 MSVC 編譯工具與 Windows SDK。
2. **CMake**：
   - 下載最新版本並安裝，安裝時記得勾選將其加入系統的 PATH 環境變數中。
3. **Ninja**：
   - 一個小型的建置系統，下載後解壓縮，並將 `ninja.exe` 所在的目錄路徑加入系統的 PATH 環境變數中。
4. **Skia 庫**：
   - Aseprite 依賴 Google 的 Skia 繪圖庫。請至 Aseprite 的 GitHub Release 頁面下載官方預先編譯好的 Skia 版本，並解壓縮到指定路徑（例如 `C:\deps\skia`）。
5. **Git**：
   - 用於複製 Aseprite 專案的原始碼。

---

## 自動編譯步驟

準備好上述工具後，就可以開始編譯了：

### 1. 複製原始碼
打開終端機（如 Git Bash 或 PowerShell），執行以下指令來下載 Aseprite 的原始碼及所有子模組：
```bash
git clone --recursive https://github.com/aseprite/aseprite.git
```

### 2. 開啟 VS 開發人員命令提示字元
在 Windows 開始功能表中，搜尋並開啟 **"Developer Command Prompt for VS 2022"**（開發人員命令提示字元）。這能確保 C++ 編譯器相關的環境變數已被正確載入。

### 3. 進行 CMake 設定
在開發人員命令提示字元中，切換到剛才複製好的 `aseprite` 目錄，建立並進入 `build` 資料夾：
```cmd
cd aseprite
mkdir build
cd build
```
接著執行 CMake 設定指令，指向您的 Skia 路徑（請根據您的實際解壓縮路徑調整，以下以放置在 `C:\deps\skia` 為例）：
```cmd
cmake -DCMAKE_BUILD_TYPE=RelWithDebInfo -DLAF_BACKEND=skia -DSKIA_DIR=C:\deps\skia -DSKIA_LIBRARY_DIR=C:\deps\skia\out\Release-x64 -DSKIA_LIBRARY=C:\deps\skia\out\Release-x64\skia.lib -G Ninja ..
```

### 4. 執行編譯
設定完成且沒有報錯後，輸入以下指令開始編譯：
```cmd
ninja aseprite
```
編譯過程需要幾分鐘的時間，請耐心等待。

### 5. 完成與執行
編譯完成後，您可以在 `build/bin/` 資料夾中找到編譯好的 `aseprite.exe`。直接點擊即可啟動您的 Aseprite！
